## *05* 使用 EXAs 进行分布式网络编程
&emsp;&emsp;*要从计算机手中夺回自主权，第一步就是学会如何使用它们。而这意味着要掌握 EXA 编程。*

### 什么是EXA？
&emsp;&emsp;EXA 是“执行代理”（EXecution Agent）的缩写。这是一个小型程序，能够通过网络在计算机之间移动，且不会中断其正在进行的任务，即使跨越现实世界中巨大的距离也是如此。

&emsp;&emsp;本世纪初，Axiom 和 TEC 等大型科技公司基于他们所称的“分布式网络编程”愿景，开发了 EXA 标准。EXA 能够无视底层硬件的差异，自由传输和处理数据，从而轻松将世界万物连接在一起。如今，它们已成为所有现代计算机网络的命脉。

### 代码、值和寄存器
&emsp;&emsp;除了在计算机网络中以实体形式存在外，每个 EXA 还包含**代码区**和**寄存器**。如果您通过 EXODUS 开发环境连接到网络，您会看到左侧为每个 EXA 的窗口，该窗口允许您编辑其代码并查看其寄存器。

&emsp;&emsp;**代码区** 这是一组指令列表，用于告诉 EXA 该做什么。它使用一种专门为 EXA 设计的特殊计算机语言编写。我们将在接下来的教程部分深入探讨这种语言。

&emsp;&emsp;**寄存器** 不妨将它们视为能够存储值的“插槽”。这些值可以是**整型**，比如 `38`、`5074` 或 `-203`，也可以是字符串，比如 `SECRET`、`TRASH WORLD NEWS` 或 `ABC123`。寄存器可以通过代码中的指令进行读写操作。

&emsp;&emsp;寄存器有不同的类型。有些寄存器仅用于存储值，因此当你读取它们时，它们会返回你上次写入的值（这些是 **X** 和 **T** 寄存器）。其他寄存器实际上是更高级功能的接口，例如读写文件（**F** 寄存器）或与其他 EXA 通信（**M** 寄存器）。不必担心一开始就要掌握所有寄存器的用法。在后续教程中，你会逐步学习如何使用不同类型的寄存器。

## 主机、链接、文件和硬件寄存器

&emsp;&emsp;以下是一些您在使用 EXA 探索网络时将会看到的内容：

### 主机
&emsp;&emsp;主机是网络中的一个实体。EXA 只能存在于主机内部。当你创建一个新的 EXA 实例时，它们会位于你的主机中，但运行时，它们可以上传到目标主机并在那里执行。

&emsp;&emsp;由于系统性能和内存的限制，所有主机同时容纳的EXA数量均受限。在 EXODUS 中，这通过主机内部的可用网格空间数量来体现。每个 EXA 占用一个网格空间。

### 链接
&emsp;&emsp;链接将主机相互连接，EXA 正是通过这些链接在主机之间移动。链接的两端都有一个数字标识符，方便您在代码中引用它们。大多数链接支持双向通行，但有些仅限单向通行，还有些则完全无法使用。

### 文件
&emsp;&emsp;文件用于存储值的序列，包括数字和关键字。它们与 EXA 有些相似，因为它们存在于主机中并占用空间，但本身不包含代码或行为。它们的唯一用途就是存储值。

&emsp;&emsp;要操作文件，你需要使用 EXA。EXA 可以创建和删除文件，向文件写入和从文件读取数据，以及通过拖放操作移动文件。与链接类似，每个文件都有一个数字 ID，让你可以在代码中引用它。

### 硬件寄存器
&emsp;&emsp;某些主机包含特殊的硬件寄存器。这些寄存器与可能连接到主机计算机的外部硬件相关联。具体情况因系统而异，因此这些寄存器有时仅支持读取，有时仅支持写入，而有时则两者兼备。这些寄存器可能包含数字或关键字。这些寄存器的名称由井号（#）后跟四个字符组成，例如 `#POWR` 或 `#ENAB`。您可以在代码中直接使用这些名称。

## *08* Ghast带你闯关（教程）
&emsp;&emsp;我坚信任何人都能学会 EXA 编程。正因如此，我搭建了一个包含一些教程的资源库，供大家尝试。即使你觉得自己不是程序员，也请试一试。说不定你会给自己带来惊喜。

### 教程 1：基础知识
&emsp;&emsp;在本教程的第一部分中，你的目标是将ID为`200`的特定文件从名为“inbox”的主机移动到名为“outbox”的主机。你将通过ID为`800`的链接进行移动。

&emsp;&emsp;启动 EXODUS 并连接到教程网络。系统将自动为您创建一个新的空白解决方案。（“解决方案”只是 EXA 文件的集合，当您点击“运行”按钮时，这些文件便会开始执行。）

&emsp;&emsp;现在，屏幕左侧会显示一个空白的EXA窗口，其中代码窗口为空白。请在 EXA 中输入以下五行代码：

``` C++
LINK 800 //通过链接800进入网络
GRAB 200 //抓起文件200
LINK 800 //由链接800从“inbox”进入“outbox”
DROP     //放下被抓起的文件
HALT     //终止 EXA
```

&emsp;&emsp;按下 **step** 按钮，程序将逐条指令执行。当你感到无聊时，可以按下 **run** 按钮，观看程序自动运行。

&emsp;&emsp;确认程序运行符合预期后，您可以点击 **fast-forward** 按钮，让程序一直运行到结束。恭喜，你刚刚编写了第一个 EXA 程序。

### 教程 2：读写文件
&emsp;&emsp;好。接下来的任务与第一个非常相似，只是你需要读写该文件，而不是仅仅移动它。请阅读 EXODUS 中的要求以获取完整说明。现在就去读一读吧。

&emsp;&emsp;要读取或写入 EXA 持有的文件，我们必须使用 **F** 寄存器。在执行操作的过程中，我们还将使用 **X** 寄存器来存储中间值。请将以下代码输入到默认的空 EXA 中：

``` C++
LINK 800   //通过链接800进入网络
GRAB 200   //抓起文件200
COPY F X   //从 F 读取数据，并将该值写入 X
ADDI X F X //计算 X + F，并将结果写入 X
MULI X F X //计算 X × F，并将结果写入 X
SUBI X F X //计算 X - F，并将结果写入 X
COPY X F   //从 X 读取数据，并将该值写入 F
LINK 800   //由链接800从“inbox”进入“outbox”
DROP       //放下被抓起的文件
HALT       //终止 EXA
```

&emsp;&emsp;当您逐步执行程序并运行 `GRAB` 指令时，请注意，文件 200 的窗口会出现在当前正在显示该文件的 EXA 窗口下方。

&emsp;&emsp;此外，请注意文件窗口中的“文件光标”，它会高亮文件中的第一个值。当 EXA 从 **F** 寄存器读取数据时，它将读取文件光标所指向的值。同样地，向 **F** 寄存器写入数据会替换文件光标所指向的值。如果文件光标位于文件末尾，则会将新值追加到文件末尾，而非替换现有值。

&emsp;&emsp;还有一点。读写 **F** 寄存器会自动将文件光标移动到文件中的下一个值。有时这很方便，有时则不然。

### 教程 3：EXA 之间的通信
&emsp;&emsp;接下来的教程涉及一个只能通过单向链接访问的文件。 ‍负责获取该文件的 EXA 将被困住，因此你需要创建两个 EXA ，并让它们相互通信。

&emsp;&emsp;这次，我希望你能独立编写代码，不过这里有些提示：

* 您可以使用 `MAKE` 指令创建一个新文件。生成的文件默认是空的，并将自动由创建它的 EXA 抓起。
* EXA 可以使用 `WIPE` 指令删除其抓起的文件。
* 两个 EXA 可以通过 **M** 寄存器相互通信。如果一个 EXA 将一个值写入 **M** 寄存器，而另一个 EXA 从 **M** 寄存器读取该值，则该值将从写入方传输至读取方。
* 如果一个 EXA 在读取 **M** 寄存器时，没有其他 EXA 在向其写入数据，或者反之亦然，它会等待，直到另一个 EXA 接通通信的另一端。你不需要精确地控制时间。
* EXODUS 用户界面中有一个切换开关，可用于将 **M** 寄存器设置为全局（global）或本地（local）模式。在全局模式下，只要两台 EXA 之间存在链接路径，它们即可在网络的任何位置进行通信。在本地模式下，两台 EXA 只有在位于同一主机时才能进行通信。最后，处于全局模式的 EXA 无法与处于本地模式的 EXA 通信，即使它们位于同一主机中也是如此。

### 教程 4：循环与条件语句
&emsp;&emsp;你的最后一项任务是创建一个包含数字序列的文件。为此，你需要使用一种叫做“**循环**”的结构，它能让你反复执行几行代码，直到满足停止条件为止。以下是一些提示：

* 您可以使用 `TEST` 指令对寄存器和/或数值进行比较，例如检查 **X** 是否等于 `38`（`TEST X = 38`）。如果测试结果为真，则 **T** 寄存器被置为1。如果测试结果为假，则 **T** 寄存器被置为 `0`。
* 单看 `TEST` 指令本身并没有太大用处，但如果将其与条件跳转指令（`TJMP` 和 `FJMP`）结合使用，就可以根据测试结果跳转到程序的不同部分。当 **T** 寄存器被置为 `1` 时，`TJMP`（真时跳转）生效；当 **T** 寄存器被置为 `0` 时，`FJMP`（假时跳转）生效。明白其中的关联了吗？你需要在程序的其他位置使用 `MARK` 指令指定跳转目标。
* 下面是一个循环的示例。假设你想将数值 `9999` 写入文件 10 次。你可以像这样实现，该方法通过 **X** 寄存器来记录循环的迭代次数：

``` C++
COPY 0 X    //将 X 寄存器重置为 0
MARK LOOP   //定义一个名为 LOOP 的标签
COPY 9999 F //将 9999 写入抓起的文件
ADDI X 1 X  //将 X 增加 1
TEST X = 10 //判断 X 是否等于 10，并将结果写入 T
FJMP LOOP   //如果 X 尚未等于 10，则跳转到 LOOP
```

* 当 `TJMP` 和 `FJMP` 指令读取 **T** 寄存器中的值时，它们并不关心该值是来自 `TEST` 指令，还是由你自己存入的。实际上，你可以将任何数据存入 **T** 寄存器，只要你接受下次执行 `TEST` 指令时该值会被清空即可。

## *12* 黑客技能：运行时错误及其利用方法
&emsp;&emsp;有时 EXA 会尝试执行一些被禁止的操作。一旦发生这种情况，该 EXA 就会被主机自动终止。从功能上讲，这与执行一条 `HALT` 指令的效果相同。虽然大多数错误都平淡无奇且毫无用处，但其中有些却能为你所用。

### 我们先从一些没用的错误开始吧：

* **除以零**

&emsp;&emsp;在电脑上不能进行除以零的运算。这是数学上的某种原理。不必太在意其中的原因，总之别这么做就行了。

* **字符串数学运算**

&emsp;&emsp;无法对字符串执行加法、减法、乘法等运算。数学运算仅限于数值。

* **访问无效的 F 寄存器**

&emsp;&emsp;如果你的 EXA 在未抓起文件时读写 **F** 寄存器，将会报错并终止运行。请务必先抓起文件。

* **访问无效的硬件寄存器**

&emsp;&emsp;与上一个类似。在尝试读取或写入硬件寄存器之前，请确保 EXA 位于与该寄存器相同的主机上。

&emsp;&emsp;在编写代码的过程中，你可能会遇到很多这类错误，并会在编写过程中逐步修复它们。不过，现在让我们来谈谈更有趣的那种错误。

&emsp;&emsp;有时你并不了解所用主机的所有细节。链接可能会改变，文件可能会消失。有些关于运行环境的信息，只有通过尝试执行代码并可能遭遇失败才能了解。这看起来或许有些杂乱无章，但你又不是在某家大型软件公司工作，在那里这种“杂乱”会招致白眼（如果你确实身处这样的环境，那我只能说声抱歉，或许你该考虑改变一下生活了）。在这里，唯一重要的是把目标完成。

### 以下是一些更易于利用的错误：

* **无效文件访问**

&emsp;&emsp;假设你想查找某个特定文件。尝试查找该文件的 EXA 程序会终止运行，但这样你就能确定该文件并不存在。

* **无效链接访问**

&emsp;&emsp;假设你正在寻找一个特定的链接。尝试访问该链接的 EXA 会终止运行，但现在你已经知道那个链接不存在了……看出规律了吗？

---

&emsp;&emsp;说真的，别害怕让你的 EXA 程序报错、崩溃、挂掉——不管你怎么称呼它。它们在你眼里可能看起来很可爱，但请记住，它们存在的意义就是不断生成、运算，然后死去。你可以在程序中利用这一点。

## *14* Axiom EXA 语言参考指南

&emsp;&emsp;EXA 虚拟机（EXA-VM）允许多个执行代理（EXA）在由主机组成的共享网络中运行。在该网络内，EXA 可以被动态创建、销毁，并从一台主机转移到另一台主机。即使多个 EXA 位于同一台主机上，EXA-VM 也能确保所有 EXA 独立且同时运行。

&emsp;&emsp;EXA 程序由一系列指令组成。每条指令需要零个或多个操作数。每条指令所需的操作数均使用简短的缩写形式指定：

```
R    寄存器
R/N  寄存器，或介于 -9999 至 9999 之间的数字
L    由 MARK 伪指令定义的标签
```

### 寄存器
&emsp;&emsp;**X** 寄存器是一个通用存储寄存器，可以存储数字或关键字。

&emsp;&emsp;**T** 寄存器是一个通用存储寄存器，可以存储数字或关键字。它也是 `TEST` 指令的目标寄存器，并作为条件跳转（`TJMP` 和 `FJMP`）的判断条件。

&emsp;&emsp;**F** 寄存器允许 EXA 读取和写入已获取文件的内容。当 EXA 获取一个文件时，其“文件光标”将定位于文件中的第一个值。读取 **F** 寄存器将读取该值；写入 **F** 寄存器将覆盖该值。读取或写入 **F** 寄存器后，文件光标将自动向前移动。写入文件末尾时，系统会追加新值而非覆盖原有内容。

&emsp;&emsp;**M** 寄存器控制着 EXA 的消息传递功能。当一个 EXA 向 **M** 寄存器写入数据时，该值将被存储在该 EXA 的出站消息槽中，直到另一个 EXA 从 **M** 寄存器读取并接收该先前写入的值。数字和关键字均可通过这种方式传输。

&emsp;&emsp;如果一个 EXA 向 **M** 寄存器写入数据，它将暂停执行，直到另一个 EXA 读取该值。如果一个 EXA 从 **M** 寄存器读取数据，它将暂停执行，直到有可读取的值。如果两个或多个 EXA 试图同时从另一个 EXA 读取数据（反之亦然），其中一个会成功，但哪个会成功是无法预测的。

&emsp;&emsp;默认情况下，一个 EXA 可以与同一网络中的任何其他 EXA 进行通信。可以通过在 EXODUS 接口中切换全局/本地设置，或执行 `MODE` 指令，将通信范围限制为同一主机内的 EXA。处于全局模式的 EXA 无法与处于本地模式的 EXA 进行通信，即使它们位于同一主机内也是如此。

### 硬件寄存器
&emsp;&emsp;某些运行 EXA-VM 的主机可能通过硬件寄存器提供对连接硬件的访问。硬件寄存器的有效名称由井号（“#”）后跟四个字符组成，例如 `#POWR` 或 `#ENAB`。根据主机的配置不同，硬件寄存器可能是可读写、仅可读或仅可写的。

`HALT`

&emsp;&emsp;终止与该 EXA 位于同一主机上的另一个 EXA，优先处理由同一用户创建的 EXA。如果有多个可能的目标，目标将以不可预测的方式被选中。

`LINK R/N`

&emsp;&emsp;访问指定 ID 的链接

&emsp;&emsp;将当前主机的名称复制到指定的寄存器中。

`MODE`

&emsp;&emsp;在全局和本地之间切换 **M** 寄存器模式。

`VOID M`

&emsp;&emsp;从 **M** 寄存器读取并丢弃一个值。

`TEST MRD`

`MAKE`

&emsp;&emsp;创建并抓起一个新文件。

&emsp;&emsp;抓起指定 ID 的文件。

&emsp;&emsp;将当前文件的 ID 复制到指定的寄存器中。

`WIPE`

&emsp;&emsp;删除当前抓起的文件。

## *18* 调试噬菌体
&emsp;&emsp;一种俗称“噬菌体”的神秘疾病，会将人体逐渐转化为计算机部件，这听起来像是科幻小说或阴谋论中的情节，但遗憾的是，现实中确实存在这种情况。在医学界，这种疾病被称为“进行性神经可塑性功能障碍”（这一名称已过时，且完全不准确），或较少使用的“Wurzner综合征”。

&emsp;&emsp;乍一看，变成一台电脑听起来似乎很酷——就像变成了半机械人一样。遗憾的是，这种噬菌体并不会将你的身体变成任何类似于正常运作的电脑的东西。如果任其发展，噬菌体最终只会让身体变成一堆毫无生气的、毫无用处的电路组合体。

&emsp;&emsp;我们认为，经常使用电脑的人更容易患上这种病，已知的所有病例都发生在每天使用电脑八小时或更长时间的人群中。这种病会在六到十八个月内从四肢（通常是双手，但偶尔也出现在双脚）向身体中心扩散，逐渐将人体细胞替换为一种含有半导体亚结构的塑料状物质所构成的连通结构。

&emsp;&emsp;目前治疗该噬菌体的最佳方案是，一旦出现病变组织就立即切除，但这显然并非理想的解决方案。目前还有其他几种疗法正在研发中。有一种超适应症口服药物似乎能有效延缓其扩散，但对大多数潜在患者而言，其价格高得令人望而却步。

&emsp;&emsp;除此之外，人们对这种噬菌体的了解仍然十分有限，因此各种猜测便争相填补这一空白。关于该噬菌体是某秘密政府实验室在研究人机接口时意外逃逸的实验体的说法无法得到证实；同样无法证实的是那些坦率地说相当荒谬的设想，例如某种神秘的“计算机大灾难”，据称碳基生命体与硅基生命体将融合在一起，形成一种更高层次的意识形态。

&emsp;&emsp;有一点我们可以通过实验验证，那就是噬菌体确实最初是被设计来作为一种促进人机直接交互的手段。作为一所知名医学院的研究生，我得以利用实验室与一位患有该病的熟人一起进行了一些“非正式”的实验。出于显而易见的原因，我们两人都将保持匿名。

### 将噬菌体节点看作 EXA 主机进行交互

&emsp;&emsp;关于噬菌体感染的组织，最有趣的一点或许在于，它似乎确实提供了可以连接到现成硬件调试器的接口。在本文所述的实验中，我使用了通常用于调试基于 EXA 的嵌入式系统的 Mitsuzen HDI-10。

&emsp;&emsp;已知噬菌体节点对神经（及类神经）细胞具有亲和力，会干扰神经活动，并引发与该疾病相关的退行性症状。进一步支持噬菌体旨在支持人机交互这一观点的是，连接至神经细胞的噬菌体节点在其对应宿主中呈现硬件寄存器，这些寄存器允许对连接神经细胞的电位进行测量、激发和抑制。单位似乎以毫伏为单位；对于不了解相关知识的人来说，神经的静息电位约为 `-70` mV，兴奋时可飙升至约 `50` mV，抑制时则可降至低至 `-120` mV。

&emsp;&emsp;“她为什么这么冷酷无情？”追悼会前几天，我问本。他停顿了一下，把那杯我冲得难喝透顶的咖啡放在了同一个早餐吧台上。他耸了耸肩。

---

&emsp;&emsp;我很想念玛丽亚，想念她给我编辫子时，一边唱歌一边用尤克里里伴奏的样子。她有一头卷发，脸上有酒窝，而且很容易晒黑。而莎伦和我则有着同样苍白的皮肤，光是想到太阳，皮肤就会起水泡。

&emsp;&emsp;玛丽亚曾抱着我，让我坐在她腿上，一边看老卡通片一边轻轻摇晃。她会喂我吃花生酱饼干，还会帮我梳头。她的头发又浓密又长，垂到腰际以下，她总是把它编成一条紧实的辫子。她是一条美人鱼吗？我至少问过她一次。她只是微笑着，咯咯地笑了起来。

&emsp;&emsp;“玛丽亚要去哪里？”

&emsp;&emsp;“玛丽亚正在组建自己的家庭。”

&emsp;&emsp;一股苦涩感瞬间涌上心头。 随后，迷茫的雾气笼罩了我的双眼。

&emsp;&emsp;“组建自己的家庭？”

&emsp;&emsp;“玛丽亚认识了一个不错的年轻人，要结婚了。”

&emsp;&emsp;“那你的年轻人呢？”

&emsp;&emsp;莎伦冲我咧嘴一笑，双手搭在我的肩膀上。 

&emsp;&emsp;“该开始学习了。”

---

&emsp;&emsp;玛丽亚走后没多久，莎伦就带回了一台亮紫色的全功能电脑。她说这是鲍威尔研究所人工智能部门送的礼物。当我打开电脑时，莎伦笑逐颜开，蹲在我身旁——那一刻，对我来说，简直就像是此生最接近圣诞早晨的时刻。电脑发出清脆的提示音，一道如浪潮般翻涌的波纹在反光屏幕上翻腾旋转。

&emsp;&emsp;“你好，” 设备里传来一个声音。“我叫乔迪。你叫什么名字？”

&emsp;&emsp;我盯着屏幕，看着我们两人的倒影。

&emsp;&emsp;“谢谢，” 我说。 

&emsp;&emsp;“你好，淘气鬼，”乔迪说道。

&emsp;&emsp;“虽然还有一些问题，但AI团队一直在进行修复。前景非常可观。我觉得你会非常喜欢拥有这样一个虚拟……朋友的。”

&emsp;&emsp;在接下来的三年里，我从未纠正过乔迪，而她一直叫我“淘气鬼”。

---

&emsp;&emsp;莎伦一直想做对我和她自己最有利的事。所以，我是由另一个女人代孕的。罗莎琳。

&emsp;&emsp;我第一次听到她的名字，是在偷听莎伦和本的谈话时。我顿时为之倾倒。我用一笔流畅的草书 “**R**” 在空中描摹她的名字，想象着她的面容，尽管我知道那与我的长相截然不同，也与莎伦的截然不同。mark

&emsp;&emsp;罗莎琳。罗莎琳。我曾想象她是一位有着一头长长的赤褐色长发和一双绿眼睛的公主。我曾想象她是一位脸颊红润、身上有纹身的离家出走者。她是一位诗人。她是一位宇航员。她是一位外科医生。她是一位舞者。她是一位创作者。她是一位母亲。

&emsp;&emsp;“乔迪，‘罗莎琳’这个名字有什么意义？”

&emsp;&emsp;“罗莎琳是威廉·莎士比亚的《罗密欧与朱丽叶》中的一位次要角色。这是罗莎琳德这个名字的变体，意为‘温顺的马’。”mark

&emsp;&emsp;一如既往，乔迪浇灭了我那天马行空的想象。

&emsp;&emsp;乔迪最终并未如研究所所期望的那样，成为首屈一指的家庭教育助手。九岁那年，莎伦向我介绍了普里娅和本，他们后来成了我所有科学、技术、工程和数学相关科目的私人导师。除此之外，没有其他科目。

&emsp;&emsp;普里娅和本每周会带着他们的笔记本电脑来三次，偶尔还会带上教科书（普里娅坚持说这是为了打好基础）。三角学和微积分由普里娅负责，她就像一座方程与变量的喷泉。本则负责生物学和化学。直到我证明了自己的数学能力后，普里娅才允许我学习物理。他们两人就像两块磁铁，总是互相调侃、互相打趣。但他们教会了我对数与算法、力学与动力学、有丝分裂与共生。

&emsp;&emsp;十四岁那年，我已经达到了私立中学的毕业要求。为了庆祝这一成就，普里娅、本和我挤进普里娅的那辆微型电动车，开车去了“乳品皇后”冰淇淋店。

&emsp;&emsp;当我们回到公寓时，莎伦打开门，冲我笑了笑。“现在我们可以开始你的真正教育了！”

&emsp;&emsp;“什么意思？” 

&emsp;&emsp;“你要去鲍威尔研究所工作了。和我一起。”

&emsp;&emsp;“和我们一起，”普里娅补充道。

&emsp;&emsp;莎伦瞥了普里娅一眼，又看了看本。

&emsp;&emsp;“你的胡子上沾了冰淇淋。”

&emsp;&emsp; 说完，她便消失在公寓昏暗的玄关里。
